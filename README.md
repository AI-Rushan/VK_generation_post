## Навигация

- Техническая версия (подробно): [README.technical.md](README.technical.md)

Веб-приложение на Flask, которое автоматически создает продающие посты по ссылке на товар, позволяет редактировать результат, сохранять посты в избранное и публиковать/планировать публикации во ВКонтакте.

Проект подходит для малого бизнеса, SMM-специалистов и владельцев магазинов, которым нужно быстро готовить контент для соцсетей без ручного написания каждого поста.

## Что умеет приложение

- Принимает URL товара и извлекает базовый контент страницы (title, meta description, текстовые фрагменты).
- Генерирует пост через LLM (ProxyAPI) с учетом выбранных параметров стиля.
- Поддерживает настройки генерации:
  - тон (дружелюбный, экспертный, энергичный),
  - длина (короткий, средний, подробный),
  - количество эмодзи,
  - стиль CTA,
  - стиль хештегов,
  - модель LLM.
- Очищает текст от служебных артефактов (названия разделов, лишняя нумерация и т.п.).
- Позволяет редактировать пост прямо в интерфейсе перед публикацией.
- Публикует пост во ВКонтакте (в сообщество) через `wall.post`.
- Поддерживает отложенную публикацию во ВКонтакте через `publish_date`.
- Сохраняет:
  - избранные посты в `favorites.json`,
  - запланированные посты в `scheduled_posts.json`.

## Как это работает

1. Пользователь вставляет ссылку на товар.
2. Бэкенд скачивает страницу и извлекает ключевую информацию.
3. Формируется промпт с параметрами стиля и данными товара.
4. Запрос отправляется в ProxyAPI (`/chat/completions`).
5. Полученный текст очищается и отображается в интерфейсе.
6. Пользователь может:
   - скопировать текст,
   - отредактировать и сохранить правки,
   - опубликовать сразу,
   - запланировать публикацию,
   - добавить в избранное.

Если страницу товара нельзя прочитать (блокировка/ошибка), приложение использует fallback-логику: берет заголовок из URL и просит модель не выдумывать факты.

## Быстрый старт

### 1) Клонирование и установка зависимостей

```bash
git clone <repo-url>
cd site_generation_post
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 2) Настройка переменных окружения

Скопируйте `.env.example` в `.env` и заполните значения:

```bash
cp .env.example .env
```

Переменные:

- `PROXYAPI_KEY` - ключ доступа к ProxyAPI.
- `VK_ACCESS_TOKEN` - токен сообщества VK с правом публикации на стену.
- `VK_GROUP_ID` - ID сообщества (только число, без `-`).

### 3) Запуск

```bash
python app.py
```

Приложение стартует на:

- `http://127.0.0.1:5500`

## Использование

1. Откройте страницу приложения.
2. Вставьте ссылку на товар.
3. Выберите настройки генерации.
4. Нажмите **"Сформировать пост"**.
5. При необходимости отредактируйте текст.
6. Выберите действие:
   - **Опубликовать пост**,
   - **Запланировать** (указать дату и время),
   - **В избранное**.

## Важные детали и ограничения

- Данные `favorites.json` и `scheduled_posts.json` хранятся локально в файлах, без БД.
- Проект ориентирован на один локальный инстанс; при многопользовательском сценарии нужна серверная БД и авторизация.
- Некоторые сайты могут блокировать парсинг; в этом случае генерация работает по ограниченным данным.
- Отложенная публикация зависит от правил и ограничений VK API.
- `debug=True` включен для разработки; для production нужен WSGI-сервер и безопасные настройки.
